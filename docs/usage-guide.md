# AI書籍自動制作ツール 使い方ガイド

## はじめに

このガイドでは、AIを使って電子書籍を自動で作る手順を説明します。

- **前提**: セットアップガイドが完了していること
- **制作時間**: 約30〜45分（ほぼ自動。操作は数回のクリックだけ）
- **費用**: ChatGPT契約内で追加費用なし

---

## 全体の流れ（5ステップ）

```
Step 1: Chrome（書籍制作用）を起動する
Step 2: コマンドプロンプトで制作を開始する
Step 3: AIの質問に答える（テーマ・著者名・タイトル確認・目次確認）
Step 4: 自動制作を見守る（原稿→メタデータ→表紙→A+画像）
Step 5: 完成データを取得する
```

---

## Step 1: Chrome（書籍制作用）を起動する

1. 普段使いのChromeが開いている場合は、**すべて閉じてください**
2. デスクトップにある **「Chrome（書籍制作用）」** をダブルクリックします
3. Chromeが開いたら、ChatGPTにログイン済みか確認します
   - ログインしていない場合は、 https://chat.openai.com/ にアクセスしてログインしてください
4. ChatGPTにログインできたら、**Chromeはそのまま開いておいてください**（閉じないでください）

---

## Step 2: コマンドプロンプトで制作を開始する

1. キーボードの **Windowsキー** を押します
2. **cmd** と入力します
3. 表示された **「コマンドプロンプト」** をクリックして開きます
4. 黒い画面が表示されたら、以下の文字を **そのままコピーして貼り付け**、Enterキーを押します

**コマンドプロンプト（cmd）の場合:**
```
cd %USERPROFILE%\repos\ebook-autowriter-codex
```

**PowerShell / Windows Terminal の場合:**
```
cd $env:USERPROFILE\repos\ebook-autowriter-codex
```

5. 続けて、以下を **そのままコピーして貼り付け**、Enterキーを押します（最新版に更新されます）

```
git pull
```

> 「Already up to date.」と表示された場合はすでに最新です。ファイル名が表示された場合は更新が完了しています。

6. 続けて、以下を **そのままコピーして貼り付け**、Enterキーを押します

```
codex --dangerously-bypass-approvals-and-sandbox "テキスト版で「{テーマ}の電子書籍を制作」を開始してください。"
```

> **図解版（図解40〜60枚入り）** で制作したい場合は、「テキスト版」を「図解版」に変えてください:
>
> ```
> codex --dangerously-bypass-approvals-and-sandbox "図解版で「{テーマ}の電子書籍を制作」を開始してください。"
> ```

7. **「Reading additional input from stdin...」** と表示されたら、準備完了です。AIが動き始めます

> **コピーの仕方**: 上の文字を左クリックしながらなぞって青く選択 → 右クリック →「コピー」。コマンドプロンプトに貼り付けるときは、黒い画面の上で右クリックすると貼り付けられます。

---

## Step 3: AIの質問に答える

AIが順番に質問してきます。コマンドプロンプトの黒い画面に直接文字を入力して、Enterキーを押して答えます。

### 質問1: テーマと著者名

AIがこのように聞いてきます:

```
電子書籍の制作を開始します。以下を教えてください。
1. テーマ（必須）
2. 著者名（必須）
3. 参考資料（任意）
```

以下のように入力してEnterを押します（例）:

```
テーマ: 40代からの腸活ダイエット
著者名: 田中太郎
```

> **ポイント**: 参考資料は入力しなくてもOKです。テーマと著者名だけで制作できます。

### 質問2: タイトルの選択

AIが10個のタイトル案を提案してきます。気に入った番号を入力してEnterを押します。

入力例:

```
8番でOK
```

> **ポイント**: 自分で考えたタイトルをそのまま入力してもOKです。

### 質問3: 目次（構成）の確認

AIが5章構成の目次案を提案してきます。そのままでよければ、以下のように入力してEnterを押します。

```
OK
```

> **ポイント**: 修正したい場合は「第3章を○○に変更して」のように入力すれば修正されます。

---

## Step 4: 自動制作を見守る

目次を承認すると、AIが以下を**すべて自動で**行います:

- 原稿執筆（25,000字）
- 出版メタデータ生成
- Kindle申請データ生成
- 表紙画像生成（Chromeを自動操作してChatGPT Images 2.0で生成）
- A+コンテンツ画像4枚生成

**途中でAIが確認を求めてきたら「OK」と入力してEnterを押してください。**

### 表紙プロンプトの確認

AIがこのように聞いてきます:

```
このプロンプトで表紙を生成してよろしいですか？
```

→ **「OK」** と入力してEnterを押します

### 表紙画像の確認

AIがChromeで自動的に表紙画像を生成します。完了すると確認を求めてきます。

→ **「OK」** と入力してEnterを押します

### A+画像の確認

AIがA+コンテンツ画像を4枚生成します。完了すると確認を求めてきます。

→ **「OK」** と入力してEnterを押します

### 完了

すべての工程が終わると、AIが以下のように表示します:

```
電子書籍の制作が完了しました
```

これで書籍の制作は完了です。

---

## Step 5: 完成データを取得する（6/2更新）

1. デスクトップにある **「BookRestore」** アイコンをダブルクリックします
   - ※ 「BookRestore」と「BookDownload」の2つがありますが、**どちらか1つだけ**クリックしてください。両方クリックする必要はありません
   - Codex CLI で制作した場合 → **BookRestore**
   - Codex Cloud（ブラウザ版）で制作した場合 → BookDownload
2. 画像が自動で復元され、完成データが入ったフォルダが自動で開きます
3. 以下のファイルが入っています

### テキストファイル（メモ帳やWordで開けます）

| ファイル名 | 内容 |
|-----------|------|
| manuscript.md | 原稿（25,000字） |
| manuscript.html | HTML原稿（ブラウザで開くとKindleのイメージが見れます） |
| research.md | AIが調べたリサーチ結果 |
| book_meta.md | タイトル・著者名 |
| listing.txt | Kindle出版用メタデータ |
| kindle_application.txt | Kindle申請用データ |
| cover_prompt.txt | 表紙のプロンプト（再生成したいときに使います） |

### 画像ファイル（images フォルダの中に入っています）

| ファイル名 | 内容 |
|-----------|------|
| cover.jpg | 表紙画像（1600×2560px） |
| aplus_1.png | A+画像1（問題提起） |
| aplus_2.png | A+画像2（共感） |
| aplus_3.png | A+画像3（解決策） |
| aplus_4.png | A+画像4（CTA） |

---

## 2冊目以降の制作

Step 1〜5をもう一度繰り返すだけです。毎回新しいテーマで制作できます。

前に作った書籍のデータが消えることはありません。テーマごとに別のフォルダに保存されます。

## よくある質問

### Q: 制作中にエラーが出ました

A: コマンドプロンプト（黒い画面）を右上の「×」で閉じて、Step 2からやり直してください。

### Q: 画像の品質が悪いです

A: ChatGPTにログインした状態でChromeが開いているか確認してください。ログインしていないと画像が低品質になることがあります。

### Q: 途中で止まってしまいました

A: 5分以上何も表示されない場合は、キーボードの **Ctrl** キーと **C** キーを同時に押してキャンセルしてください。その後、Step 2からやり直してください。

### Q: 表紙のデザインを変えたいです

A: Step 4の「表紙プロンプトの確認」のときに、「もっとシンプルにしてください」「色を青系にしてください」のように修正の指示を入力できます。

### Q: 原稿の文章を修正したいです

A: 完成フォルダの中にある manuscript.md をメモ帳やWordで開いて、直接編集できます。

### Q: 2冊目を作ったら前のデータは上書きされますか？

A: 上書きされません。テーマごとに別のフォルダ（output/テーマ名/）に保存されるので、前のデータはそのまま残ります。
