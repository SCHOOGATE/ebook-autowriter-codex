# AI書籍自動制作ツール セットアップガイド

---

## このガイドでできるようになること

- テーマを入力するだけで、25,000字の電子書籍原稿が自動で作られます
- 高品質な表紙画像・A+コンテンツ画像も自動で作られます
- Kindle出版に必要なデータが全て揃います

---

## 必要なもの（事前準備）

以下の4つを用意してください。

| 必要なもの | 補足 |
|-----------|------|
| Windows または Mac のパソコン | どちらでも動作します（本ガイドはWindows向けの手順です。Mac版は別途ご案内） |
| インターネット接続 | Wi-Fi でも有線でもOK |
| ChatGPT Plus（月額$20）または Pro（月額$200）のアカウント | まだの方は https://chat.openai.com/ で登録してください |
| Google Chrome ブラウザ | まだの方は https://www.google.com/chrome/ からインストールしてください |

## 費用について

- ChatGPT Plus $20/月 または Pro $200/月（既に契約済みなら追加費用なし）
- APIクレジットやクレジットカードの追加登録は一切不要です
- このガイドでインストールするソフトは全て無料です

## Step 1: Node.js のインストール

**Node.js（ノードジェイエス）とは:** このあとインストールする「Codex CLI」というツールを動かすために必要なソフトです。裏方として動くだけなので、普段意識する必要はありません。

### 手順

1. ブラウザで **https://nodejs.org/** を開きます

2. ページの中央あたりに、緑色の大きなボタンが2つ並んでいます。**左側の「LTS（推奨版）」と書かれたボタン**をクリックします
   > [スクリーンショット撮影ポイント] 緑色のLTSボタンが表示された画面を撮影

3. ダウンロードが始まります。画面の左下（または上部）に「node-v20.xx.x-x64.msi」のようなファイル名が表示されます

4. ダウンロードが完了したら、そのファイルをダブルクリックして開きます

5. インストール画面が表示されます。以下の順にボタンをクリックしていきます:
   - 「Next」をクリック
   - ライセンス同意画面 → チェックを入れて「Next」をクリック
   - インストール先の画面 → そのまま「Next」をクリック
   - カスタムセットアップ画面 → そのまま「Next」をクリック
   - 「Install」をクリック
   - 「このアプリがデバイスに変更を加えることを許可しますか？」→「はい」をクリック

6. インストールが進みます。完了画面が出たら「Finish」をクリック
   > [スクリーンショット撮影ポイント] 「Finish」ボタンが表示された完了画面を撮影

### 正しくインストールできたか確認する方法

1. キーボードの左下にある **「Windows キー」** （窓のマークのキー）を押します
2. そのまま **cmd** と入力します
3. 検索結果に「コマンドプロンプト」と表示されるので、クリックして開きます
   > [スクリーンショット撮影ポイント] コマンドプロンプトの黒い画面が開いた状態を撮影

4. 黒い画面に以下の文字を入力して、**Enter キー**を押します:

```
node --version
```

5. **「v20.xx.x」** のように「v」から始まる数字が表示されれば成功です
   > [スクリーンショット撮影ポイント] バージョン番号が表示された画面を撮影

**うまくいかない場合:** パソコンを一度再起動してから、もう一度手順4から試してください。

## Step 2: Git のインストール

**Git（ギット）とは:** 書籍制作に必要なファイルをインターネットからダウンロードするために使うソフトです。こちらも裏方として動くだけです。

### 手順

1. ブラウザで **https://git-scm.com/** を開きます

2. 画面右側にある **「Download for Windows」** というボタンをクリックします
   > [スクリーンショット撮影ポイント] ダウンロードボタンが表示された画面を撮影

3. ダウンロードページが表示されます。「**Standalone Installer**」の「**64-bit Git for Windows Setup**」をクリックします

4. ダウンロードが完了したら、ファイルをダブルクリックして開きます

5. インストール画面が表示されます。**設定は一切変更せず**、以下の操作だけ繰り返します:
   - 「Next」をクリック（何度か繰り返します。全部で約10画面あります）
   - 最後に「Install」をクリック
   - 「このアプリがデバイスに変更を加えることを許可しますか？」→「はい」をクリック

6. インストールが進みます。完了画面が出たら「Finish」をクリック
   > [スクリーンショット撮影ポイント] 「Finish」ボタンが表示された完了画面を撮影

### 正しくインストールできたか確認する方法

1. Step 1 で開いたコマンドプロンプトがまだ開いていればそのまま使います。閉じてしまった場合は、Windows キー → cmd → コマンドプロンプトを開きます

2. 以下を入力して Enter キーを押します:

```
git --version
```

3. **「git version 2.xx.x.windows.x」** のように表示されれば成功です
   > [スクリーンショット撮影ポイント] バージョン番号が表示された画面を撮影

**うまくいかない場合:** パソコンを一度再起動してから、もう一度試してください。

## Step 3: Python のインストール

**Python（パイソン）とは:** 画像の復元処理（Base64エンコードされた画像データをファイルに変換する処理）に使うソフトです。

### 手順

1. ブラウザで **https://www.python.org/downloads/** を開きます

2. ページ上部にある黄色い **「Download Python 3.xx.x」** ボタンをクリックします
   > [スクリーンショット撮影ポイント] 黄色いダウンロードボタンが表示された画面を撮影

3. ダウンロードが完了したら、ファイルをダブルクリックして開きます

4. **ここが最も重要な手順です。** インストール画面の一番下に、以下の2つのチェックボックスがあります:
   - 「Use admin privileges when installing py.exe」
   - **「Add python.exe to PATH」**

   **必ず「Add python.exe to PATH」にチェックを入れてください。** チェックボックスをクリックすると、四角の中にチェックマークが入ります。
   > [スクリーンショット撮影ポイント] 「Add python.exe to PATH」にチェックが入った状態の画面を撮影（最重要）

5. チェックを入れたら、**「Install Now」** をクリックします

6. 「このアプリがデバイスに変更を加えることを許可しますか？」→「はい」をクリック

7. インストールが進みます。完了画面が出たら「Close」をクリック
   > [スクリーンショット撮影ポイント] 完了画面を撮影

### 正しくインストールできたか確認する方法

1. **コマンドプロンプトを一度閉じて、新しく開き直してください**（Windows キー → cmd → コマンドプロンプト）。開き直さないと Python が認識されません。

2. 以下を入力して Enter キーを押します:

```
python --version
```

3. **「Python 3.xx.x」** のように表示されれば成功です
   > [スクリーンショット撮影ポイント] バージョン番号が表示された画面を撮影

**うまくいかない場合:** 手順4で「Add python.exe to PATH」にチェックを入れ忘れた可能性があります。一度Pythonをアンインストール（「設定」→「アプリ」→「Python」→「アンインストール」）してから、手順1からやり直してください。

## Step 4: Codex CLI のインストール

**Codex CLI（コーデックス シーエルアイ）とは:** テーマを入力するだけでAIが書籍を自動制作してくれる、このツールの本体です。

### 手順

1. コマンドプロンプトを開きます（Windows キー → cmd → コマンドプロンプト）

2. 以下を入力して Enter キーを押します:

```
npm install -g @openai/codex
```

3. 画面に文字がたくさん流れます。これはインストールが進んでいる様子です。**数分かかることがあるので、そのまま待ってください。** 途中で画面を閉じないでください。

4. 最後に「added xx packages」のような表示が出たら完了です
   > [スクリーンショット撮影ポイント] インストール完了の表示を撮影

### 正しくインストールできたか確認する方法

コマンドプロンプトで以下を入力して Enter キーを押します:

```
codex --version
```

**「0.xxx.0」** のような数字が表示されれば成功です
> [スクリーンショット撮影ポイント] バージョン番号が表示された画面を撮影

**うまくいかない場合:** 「'codex' は、内部コマンドまたは外部コマンド...として認識されていません」と出る場合は、コマンドプロンプトを閉じて開き直してから再試行してください。それでもダメな場合はパソコンを再起動してください。

## Step 5: Codex CLI にログイン

ChatGPT のアカウントを使ってログインします。API キーの発行やクレジットカードの追加登録は一切不要です。

### 手順

1. コマンドプロンプトで以下を入力して Enter キーを押します:

```
codex login
```

2. ブラウザが自動的に開きます。ChatGPT のログイン画面が表示されます
   > [スクリーンショット撮影ポイント] ブラウザに表示されたログイン画面を撮影

3. いつも ChatGPT にログインしている方法でログインします:
   - メールアドレスとパスワードで入る方法
   - 「Continue with Google」で入る方法
   - どちらでもOKです

4. ログインに成功すると、ブラウザに **「Signed in to Codex」** または **「You can close this window」** のようなメッセージが表示されます。ブラウザのこのタブは閉じてOKです

5. コマンドプロンプトに戻ると、**「Successfully logged in」** と表示されています
   > [スクリーンショット撮影ポイント] 「Successfully logged in」が表示されたコマンドプロンプトを撮影

**うまくいかない場合:**
- ブラウザが自動で開かない → コマンドプロンプトに表示されたURL（https://...）をコピーして、手動でブラウザのアドレスバーに貼り付けてください
- ChatGPT にログインできない → ChatGPT Plus または Pro の有効なアカウントが必要です。https://chat.openai.com/ で契約状況を確認してください

## Step 6: Chrome をデバッグモードで起動する設定

**なぜ必要？:** Codex CLI が Chrome ブラウザを自動操作して、ChatGPT の画像生成機能（Images 2.0）を使い、表紙画像や本文画像を自動で作成します。そのために、特別な起動方法が必要です。

### ショートカットの作成手順

1. まず、あなたのパソコンの**ユーザー名**を確認します。コマンドプロンプトで以下を入力して Enter:

```
echo %USERNAME%
```

表示された文字列（例: user、tanaka、daigoなど）をメモしてください
> [スクリーンショット撮影ポイント] ユーザー名が表示された画面を撮影

2. デスクトップの何もないところで**右クリック**します

3. メニューから **「新規作成」** → **「ショートカット」** をクリックします
   > [スクリーンショット撮影ポイント] 右クリックメニューを撮影

4. 「項目の場所を入力してください」という入力欄が表示されます。以下のテキストをコピーして貼り付けます:

```
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\Users\あなたのユーザー名\ChromeDebug"
```

**重要:** `あなたのユーザー名` の部分を、手順1で確認した実際のユーザー名に置き換えてください。

例: ユーザー名が「tanaka」なら:
```
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\Users\tanaka\ChromeDebug"
```

> [スクリーンショット撮影ポイント] 入力欄にパスを入力した状態を撮影

5. **「次へ」** をクリック

6. ショートカットの名前を **「Chrome（書籍制作用）」** に変更して、**「完了」** をクリック

7. デスクトップに新しいショートカットが作成されます
   > [スクリーンショット撮影ポイント] デスクトップに作成されたショートカットを撮影

### 動作確認

1. **普段使っている Chrome が開いている場合は、全て閉じてください**

2. デスクトップの **「Chrome（書籍制作用）」** をダブルクリックして起動します

3. Chrome が開きます。初回は何もない新しい Chrome ウィンドウが表示されます（ブックマークや履歴がない状態が正常です）

4. アドレスバーに **https://chat.openai.com/** と入力して、ChatGPT にログインしておきます（初回のみ必要。2回目以降は自動ログインされます）
   > [スクリーンショット撮影ポイント] ChatGPT にログインした状態の画面を撮影

**注意事項:**
- 書籍制作の前には、必ずこのショートカットから Chrome を起動してください
- 普段使いの Chrome と同時に起動することはできません。普段の Chrome は閉じてからこちらを起動してください

## Step 7: 書籍制作リポジトリのダウンロード

書籍制作に必要なファイル一式をダウンロードします。

### 手順

1. コマンドプロンプトを開きます（Windows キー → cmd → コマンドプロンプト）

2. まず、ファイルを保存するフォルダを作ります。以下を入力して Enter:

```
mkdir %USERPROFILE%\repos
```

（「既に存在します」というエラーが出ても問題ありません。そのまま次に進んでください）

3. 次に、そのフォルダに移動します:

```
cd %USERPROFILE%\repos
```

4. ファイルをダウンロードします:

```
git clone https://github.com/SCHOOGATE/ebook-autowriter-codex.git
```

5. 画面に文字が流れ、**「done.」** のような表示が出たら完了です
   > [スクリーンショット撮影ポイント] ダウンロード完了の画面を撮影

6. ダウンロードしたフォルダに移動します:

```
cd ebook-autowriter-codex
```

### 確認方法

以下を入力して Enter:

```
dir
```

ファイルやフォルダの一覧が表示されれば成功です。
> [スクリーンショット撮影ポイント] ファイル一覧が表示された画面を撮影

## Step 8: Chrome DevTools MCP の追加

**Chrome DevTools MCP とは:** Codex CLI が Chrome ブラウザを操作するための「橋渡し」をする仕組みです。この設定をすることで、画像の自動生成が可能になります。

### 手順

1. コマンドプロンプトで以下を入力して Enter:

```
codex mcp add chrome-devtools -- npx -y @anthropic-ai/chrome-devtools-mcp@latest
```

2. **「Added global MCP server 'chrome-devtools'.」** と表示されれば成功です
   > [スクリーンショット撮影ポイント] 成功メッセージが表示された画面を撮影

**うまくいかない場合:** 「codex が認識されない」と出る場合は、Step 4 の Codex CLI インストールをもう一度実行してください。

## Step 9: デスクトップにアイコンを作成（6/2更新）

書籍が完成した後、ワンクリックでデータを取得するためのアイコンを2つ作ります。

| アイコン名 | 用途 | いつ使う？ |
|-----------|------|----------|
| **BookRestore** | 画像を復元してフォルダを開く | **Codex CLI で書籍制作した後（通常はこちらを使います）** |
| BookDownload | GitHubからデータ取得＋画像復元 | Codex Cloud で制作した場合のみ |

### 手順

1. コマンドプロンプトで以下を入力して Enter:

```
cd %USERPROFILE%\repos\ebook-autowriter-codex
```

2. 続けて以下を入力して Enter:

```
powershell -ExecutionPolicy Bypass -File scripts\create_shortcuts.ps1
```

3. デスクトップに **「BookRestore」** と **「BookDownload」** の2つのアイコンが作成されます
   > [スクリーンショット撮影ポイント] デスクトップに作成された2つのアイコンを撮影

### 使い方（どちらか1つだけクリックしてください。両方クリックする必要はありません）

**Codex CLI で書籍を作った場合（通常の使い方）:**
1. Codex CLI での制作が完了する
2. デスクトップの **「BookRestore」をダブルクリック**
3. 画像が自動で復元され、フォルダが開く
4. 原稿・表紙・A+画像がすべて見れる

**Codex Cloud（ブラウザ版）で書籍を作った場合:**
1. GitHub で Merge を完了する
2. デスクトップの **「BookDownload」をダブルクリック**
3. GitHubから最新データを取得し、画像が復元され、フォルダが開く

**うまくいかない場合:** 「スクリプトが見つかりません」と出る場合は、Step 7 のダウンロードが正しくできているか確認してください。

## トラブルシューティング（困ったときの対処法）

### よくある問題と解決方法

| 症状 | 原因と対処法 |
|------|-------------|
| 「'node' は、内部コマンドまたは外部コマンド、操作可能なプログラムまたはバッチ ファイルとして認識されていません。」 | Node.js のインストール後にパソコンを再起動していない可能性があります。パソコンを再起動してからもう一度試してください。 |
| 「'python' は認識されていません」 | Step 3 の手順4 で「Add python.exe to PATH」にチェックを入れ忘れた可能性があります。Python をアンインストールしてから、チェックを忘れずに入れて再インストールしてください。 |
| 「'codex' は認識されていません」 | コマンドプロンプトを閉じて開き直してください。それでもダメならパソコンを再起動後、Step 4 をもう一度実行してください。 |
| `codex login` でブラウザが開かない | コマンドプロンプトに表示された URL をコピーして、手動でブラウザのアドレスバーに貼り付けてアクセスしてください。 |
| 「Successfully logged in」が表示されない | ChatGPT Plus または Pro の有効なアカウントでログインし直してください。無料アカウントでは使えません。 |
| Chrome（書籍制作用）が起動しない | ショートカットの「項目の場所」を右クリック→プロパティで確認してください。ユーザー名が正しいか、普段の Chrome が閉じているか確認してください。 |
| `git clone` でエラーが出る | インターネット接続を確認してください。Wi-Fi が切れていないか、VPN を使っている場合はオフにして試してください。 |
| 全体的にうまくいかない | 全てのコマンドプロンプトを閉じ、パソコンを再起動してから、最初からやり直してください。 |

### それでも解決しない場合

以下の情報をまとめて、サポートに連絡してください:

1. どの Step でエラーが出たか
2. コマンドプロンプトに表示されたエラーメッセージ（画面のスクリーンショット）
3. パソコンの Windows バージョン（「設定」→「システム」→「バージョン情報」で確認できます）
