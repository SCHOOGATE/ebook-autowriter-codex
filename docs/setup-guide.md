## 1. はじめに読んでください

### このツールでできること

「スタート」と入力するだけで、AIが自動的に電子書籍を1冊まるごと作ってくれます。

作ってくれるもの：
- 本文（25,000字以上の原稿）
- 表紙画像
- Amazon出品用のデータ（タイトル案、キーワード、紹介文など）
- 商品ページ用の画像4枚

### 必要なもの

- パソコン（Windows でも Mac でも OK）
- インターネット接続
- クレジットカード（ChatGPTの月額料金の支払い用）

### かかる費用

- ChatGPT Plus プラン：月額 $20（約3,000円）
- GitHub：無料
- このツール：無料

### セットアップにかかる時間

初回のセットアップは約30分です。
一度設定すれば、あとは「スタート」と入力するだけで何冊でも作れます。

## 2. ChatGPTのアカウントを作る

すでにChatGPTのアカウントをお持ちの方は、次のステップ3に進んでください。

### やること

1. パソコンのブラウザ（Chrome、Edge、Safariなど）を開きます

2. アドレスバーに以下を入力して、Enterキーを押します
   https://chatgpt.com/

3. 画面に「サインアップ」（Sign up）というボタンが表示されます。クリックしてください
   ※ 日本語で「登録する」と表示される場合もあります

4. メールアドレスを入力します
   ※ 普段使っているメールアドレスでOKです

5. パスワードを決めて入力します
   ※ 8文字以上で、忘れないようにメモしておきましょう

6. 「続ける」（Continue）をクリックします

7. 入力したメールアドレスに確認メールが届きます。メールを開いて「メールアドレスを確認」ボタンをクリックしてください

8. 名前を入力する画面が出たら、ニックネームでもOKです。入力して「続ける」をクリック

これでChatGPTのアカウントが作成されました。

## 3. ChatGPTを有料プランにする

Codexを使うには、ChatGPTの有料プラン（Plus：月額$20）が必要です。

### やること

1. ChatGPTにログインした状態で、画面の左下にある自分のアイコンまたは名前をクリックします

2. メニューが表示されるので「マイプラン」（My plan）をクリックします
   ※「設定」（Settings）→「サブスクリプション」（Subscription）の場合もあります

3. プランの一覧が表示されます。**「Plus」（月額$20）** の「アップグレード」ボタンをクリックします
   ※ 英語表示の場合は「Upgrade to Plus」と表示されます
   ※ 最初はPlusで十分です。たくさん使う方はPro（$200/月）もあります

4. クレジットカード情報を入力します
   - カード番号
   - 有効期限
   - セキュリティコード（カード裏面の3桁の数字）

5. 「購入する」（Subscribe）をクリックします

6. 「アップグレードが完了しました」と表示されたら成功です

### 確認方法

ChatGPTの画面上部のメニューに「Codex」という文字が表示されていれば、有料プランが有効になっています。

※ 表示されるまで数分かかることがあります。表示されない場合はページを再読み込み（F5キー）してみてください。

## 4. GitHubのアカウントを作る

GitHubとは「ファイルの保管庫」のようなサービスです。AIが作った原稿を保存する場所として使います。
無料で使えます。

### やること

1. ブラウザで以下を開きます
   https://github.com/

2. 画面右上の「Sign up」ボタンをクリックします

3. メールアドレスを入力して「Continue」をクリック
   ※ ChatGPTと同じメールアドレスでもOK、別でもOK

4. パスワードを決めて入力し「Continue」をクリック
   ※ 15文字以上、または8文字以上で数字と小文字を含む必要があります

5. ユーザー名を決めて入力し「Continue」をクリック
   ※ 英数字とハイフン（-）が使えます
   ※ 例：taro-kindle、hanako2026 など
   ※ あとから変更もできるので、気軽に決めてOKです

6. メールの受信設定を聞かれたら「n」を入力（お知らせ不要の場合）

7. パズル認証（ロボットでないことの確認）が出たら、指示に従って操作します

8. 「Create account」ボタンをクリック

9. メールに届いた確認コード（8桁の数字）を入力します

これでGitHubのアカウントが作成されました。

## 5. ツールを自分のGitHubにコピーする（Fork）

「Fork（フォーク）」とは、誰かが作ったツールを自分のアカウントにコピーすることです。
コピーしたものは自分専用なので、自由に使えます。

### やること

1. GitHubにログインした状態で、以下のURLを開きます
   https://github.com/SCHOOGATE/ebook-autowriter-codex

2. 画面の右上あたりに「Fork」というボタンがあります。クリックしてください
   ※ 星マーク（Star）の隣にあります

3. 「Create a new fork」という画面が表示されます

4. そのまま何も変更せず、緑色の「Create fork」ボタンをクリック

5. 数秒待つと、自分のアカウントにコピーが作成されます
   ※ 画面左上が「SCHOOGATE / ebook-autowriter-codex」から「あなたのユーザー名 / ebook-autowriter-codex」に変わっていれば成功です

これでツールのコピーが完了しました。

## 6. CodexとGitHubをつなぐ

ここが一番つまずきやすいポイントです。ゆっくり一つずつ進めてください。

### 6-1. Codex Cloud を開く

1. ブラウザで ChatGPT にログインします
   https://chatgpt.com/

2. 画面の上部にメニューが並んでいます。その中から **「Codex」** をクリックします
   ※ 英語でも日本語でも「Codex」と表示されています
   ※ もし「Codex」が見つからない場合は、ステップ3（有料プラン）が完了しているか確認してください

3. 「次は何をコーディングしましょうか？」（英語：What would you like to code next?）という画面が表示されます。これがCodex Cloud です

### 6-2. 設定画面を開く

1. Codex Cloud の画面の **右上** を見てください

2. **歯車のマーク（⚙）** があります。これが「設定」ボタンです。クリックしてください
   ※ 英語では「Settings」と表示されることもあります

3. 設定画面が開きます

### 6-3. 「環境」メニューを選ぶ

1. 設定画面の **左側にメニューの一覧** が縦に並んでいます：

   日本語の場合：
   - 一般的に ← これではありません
   - **環境** ← ★ これをクリック
   - コードレビュー
   - コネクター
   - アナリティクス
   - データ コントロール
   - アクセストークン

   英語の場合：
   - General ← これではありません
   - **Environments** ← ★ これをクリック
   - Code Review
   - Connectors
   - Analytics
   - Data Control
   - Access Token

2. **「環境」（Environments）** をクリックしてください

### 6-4. リポジトリを接続する

1. 画面の中央付近に **「リポジトリを接続」** というボタンがあります
   ※ 英語の場合は **「Connect repository」** と表示されます
   ※ クリックしてください

2. GitHub の認証画面が表示されます
   - 「GitHubにログインしてください」と表示されたら、GitHubのユーザー名とパスワードを入力してログイン
   - すでにログイン済みの場合は、この画面はスキップされます

3. 「Authorize OpenAI」（OpenAIを承認する）という画面が表示されたら、緑色のボタンをクリックしてください
   ※ これはGitHubが「OpenAIにリポジトリへのアクセスを許可しますか？」と聞いています。「はい」の意味です

4. あなたのGitHubにあるリポジトリ（ファイル保管庫）の一覧が表示されます

5. 一覧の中から **「ebook-autowriter-codex」** を見つけてクリックしてください
   ※ ステップ5でForkしたものです

6. **「Connect」（接続）** ボタンをクリック

### 6-5. 接続できたか確認する

以下の3つを確認してください：

✅ 環境画面に「ebook-autowriter-codex」というリポジトリ名が表示されている
✅ Codex Cloud のトップ画面に戻ると、入力欄の下にリポジトリ名が小さく表示されている
✅ 入力欄に文字を入力できる状態になっている

3つとも確認できたら、接続成功です。おめでとうございます！

### 6-6. うまくいかないときは

**「環境」メニューが見つからない：**
→ 設定画面の左側のメニューを確認。「一般的に」（General）ではなく「環境」（Environments）です

**「リポジトリを接続」ボタンが見つからない：**
→ 「環境」（Environments）のページにあります。「一般的に」（General）のページにはありません

**リポジトリ一覧に「ebook-autowriter-codex」がない：**
→ ステップ5のForkが完了しているか確認してください
→ GitHubに別タブでログインして、自分のリポジトリ一覧に表示されるか確認してください

**認証画面でエラーが出る：**
→ ブラウザの別タブでGitHub（github.com）にログインできるか試してください
→ ブラウザのポップアップブロックをオフにしてからやり直してください

**それでも解決しない場合：**
→ スクリーンショットを撮って、AI出版ラボのコミュニティで質問してください

## 7. 電子書籍を作ってみる

セットアップ完了です！さっそく電子書籍を作ってみましょう。

### 7-1. Codex Cloud を開く

1. ChatGPTにログイン → 上部メニューの「Codex」をクリック

### 7-2. 「スタート」と入力する

1. 画面中央の入力欄に **「スタート」** と入力します

2. 入力欄の下にリポジトリ名「ebook-autowriter-codex」が表示されていることを確認

3. 送信ボタン（右端の矢印アイコン）をクリック、またはEnterキーを押します

### 7-3. テーマと著者名を入力する

Codexが質問してきます：

```
テーマ（必須）: どんな内容の本ですか？
著者名（必須）: Kindle出版時の著者名
参考資料（任意）: URL、ファイル、テキストなど
```

以下のように回答してください：

```
テーマ（必須）: ○○○○（あなたの書きたいテーマ）
著者名（必須）: ○○○○（ペンネームでもOK）
```

※ 参考資料は「なし」でもOKです。AIが自動でリサーチしてくれます。

### 7-4. 確認ポイントで判断する

制作途中で、AIが何度か「確認してください」と聞いてきます。
内容を確認して「OK」と入力すれば次に進みます。
修正したい場合は、具体的に伝えてください（例：「タイトルの3番を採用して」）。

### 7-5. 完成を待つ

全部で約15〜30分かかります。完了すると「制作が完了しました」と表示されます。

### 7-6. 成果物を確認する

完成すると以下のファイルが作られます：

- manuscript.md — Markdown原稿（25,000字以上）
- manuscript.html — HTML原稿（Word変換用）
- research.md — リサーチ結果
- book_meta.md — 書籍メタ情報（タイトル、著者名など）
- listing.txt — Amazon出品用データ
- kindle_application.txt — Kindle申請データ
- images/cover.jpg — 表紙画像
- images/aplus_1〜4.png — 商品ページ用画像4枚

## 8. 作った原稿をWordに変換する

Codexが作った原稿は「HTML」という形式です。
Kindleに出版するには「Word（DOCX）」形式に変換する必要があります。

### 方法1: 簡単な方法（パソコンが苦手な方向け）

1. まず manuscript.html をブラウザで開きます
   - パソコンのフォルダから manuscript.html を見つけます
   - ファイルを右クリック →「プログラムから開く」→ Chrome（またはEdge等）を選択
   - ブラウザに原稿が表示されます（見出しや太字もきれいに表示されます）

2. ブラウザ上で全選択してコピーします
   - ブラウザの画面をクリックしてから Ctrl+A（全選択）
   - Ctrl+C（コピー）
   - Mac の場合は Command+A → Command+C

3. Wordを開いて貼り付けます
   - Wordを開いて新しい文書を作成
   - Ctrl+V（貼り付け）
   - Mac の場合は Command+V

4. 名前を付けて保存します（.docx形式）

**この方法で保持されるもの：**
- 章タイトル・節タイトルの見出し
- 太字（強調）
- 箇条書き
- ポイントボックス

**この方法では保持されないもの：**
- 改ページ（章ごとのページ区切り）→ Wordで手動で挿入してください
  手順：章タイトルの前にカーソルを置く → 「挿入」→「ページ区切り」
- 一部のデザイン装飾（背景色など）

※ 改ページを手動で入れるのが面倒な方は、方法2をお試しください（自動で入ります）

### 方法2: ツールを使う方法（高品質にしたい方向け）

パソコンにツールをインストールして変換します。

**必要なツール（初回のみインストール）:**

1. Python — https://www.python.org/downloads/
   → 「Download Python 3.x」をクリックしてインストール
   → インストール時に **「Add Python to PATH」にチェックを入れてから** Installをクリック

2. pandoc — https://pandoc.org/installing.html
   → お使いのOS用のインストーラーをダウンロードして実行

3. 追加パッケージ — コマンドプロンプト（ターミナル）で以下を実行：
   ```
   pip install python-docx Pillow
   ```

**変換手順:**

コマンドプロンプト（Windows）またはターミナル（Mac）で以下を1行ずつ実行：

```
cd Desktop/ebook-autowriter-codex
git pull
python scripts/clean_invisible.py output/{テーマのフォルダ名}/
pandoc output/{テーマのフォルダ名}/manuscript.html -o output/{テーマのフォルダ名}/manuscript.docx
python scripts/docx_postprocess.py output/{テーマのフォルダ名}/manuscript.docx
python scripts/resize_aplus.py output/{テーマのフォルダ名}/images/
```

※ {テーマのフォルダ名} は実際のフォルダ名に置き換えてください（例：ai-side-hustle）

完成した manuscript.docx をWordで開いて内容を確認してください。

## 9. Kindleに出版する

### 9-1. KDP（Kindle Direct Publishing）にログイン

1. https://kdp.amazon.co.jp/ を開く
2. Amazonアカウントでログイン（なければ新規作成）
3. 「本棚」画面を開く

### 9-2. 新しい本を登録する

1. 「電子書籍または有料マンガ」をクリック

2. 以下の情報を入力します（Codexが作ったファイルからコピペ）：
   - 本のタイトル → book_meta.md のタイトル
   - サブタイトル → book_meta.md のサブタイトル
   - 著者名 → book_meta.md の著者名
   - 内容紹介 → listing.txt の紹介文
   - キーワード → kindle_application.txt のキーワード（7つまで）
   - カテゴリ → kindle_application.txt の推奨カテゴリ

### 9-3. 原稿と表紙をアップロード

1. 「電子書籍の原稿をアップロード」→ manuscript.docx を選択
2. 「表紙をアップロード」→ images/cover.jpg を選択
3. プレビューで内容を確認

### 9-4. 価格を設定して出版

1. ロイヤリティ 70% を選択
2. 価格を設定（kindle_application.txt の推奨価格を参考に）
3. 「Kindle本を出版」をクリック

出版後、審査に通常24〜72時間かかります。

## 10. よくある質問

**Q: Codex の利用に追加料金はかかりますか？**
A: いいえ。ChatGPT Plus（月額$20）に含まれています。追加料金はかかりません。

**Q: 1冊作るのにどれくらい時間がかかりますか？**
A: 約15〜30分です。「スタート」と入力してから完成まで、ほぼ待つだけです。

**Q: 何冊でも作れますか？**
A: はい。テーマを変えて「スタート」と入力すれば何冊でも作れます。

**Q: 参考資料がなくても大丈夫ですか？**
A: はい。テーマだけ入力すれば、AIが自動でリサーチしてくれます。自分のブログ記事やメモがあれば、より良い本になります。

**Q: 作った本の内容を自分で修正してもいいですか？**
A: もちろんです。Word形式に変換した後、自由に編集できます。特に「はじめに」「おわりに」は自分の言葉で書き換えるのがおすすめです。

**Q: 表紙が気に入らない場合は？**
A: 制作中に「この表紙で良いですか？」と聞かれるタイミングがあります。そこで「もっと明るい色にして」などと伝えれば作り直してくれます。

**Q: エラーで止まってしまったら？**
A: 入力欄に「続けて」と入力してみてください。それでもダメな場合は、新しく「スタート」からやり直せます。

**Q: 生成した書籍は他の人に見られますか？**
A: いいえ。生成物はあなたのCodex環境内にだけ保存されます。GitHubの共有リポジトリには含まれません。

**Q: パソコンがWindowsでもMacでも使えますか？**
A: はい。ブラウザが動けばどのパソコンでも使えます。
